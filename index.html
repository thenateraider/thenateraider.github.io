    Myocardial Infarction AI Prediction (EKG)

    IoT & Electronics Design

    This project involves developing a system to predict myocardial infarction and analyze 12-lead electrocardiogram (ECG) waveforms. The system operates offline on a Raspberry Pi CM5, which receives ECG data via Bluetooth from an Arduino Pro Mini 3.3V connected to an ADS1293 sensor board. The Raspberry Pi CM5 acts as the central hub, communicating with both the Arduino and the mobile application.

    AI Model Development

    The MI-detection model uses single-lead ECG (Lead II) signals to classify Myocardial Infarction (MI) versus non-MI (healthy or other conditions). Raw ECG signals are first preprocessed with Fourier Decomposition Method (FDM) to remove baseline wander and power-line interference. The cleaned signals are then decomposed into Fourier Intrinsic Band Functions (FIBFs) to isolate frequency components relevant to MI. From these, statistical and signal features—including entropy, kurtosis, energy, peak frequency, RMS, skewness, and wavelet-based features—are extracted. Feature selection is applied to retain only the most discriminative features for MI detection. The final classification is performed using machine learning algorithms such as k-Nearest Neighbors (kNN) or Support Vector Machines (SVM). For training, the system reads .hea metadata files to identify MI or Healthy labels and loads the corresponding ECG data from .dat files. Only records labeled as MI or Healthy are included in the training set, ensuring the model focuses on distinguishing MI cases. The selected features are standardized using a Scaler, then used to train the classifier, which is later applied in real-time detection with buffered ECG signals.

  • Pi OCR with Text To Speech

    Pi OCR with Text To Speech

    IoT & Electronics Design

    This Project is designed to create a portable Optical Character Recognition (OCR) system for the visually impaired. It uses a Raspberry Pi Zero 2W as its core, paired with a Pi AI Camera and a speaker, to capture images, process them, and then audibly read out any detected text. The system's primary principle is to provide a simple, button-activated tool that converts printed text into spoken words, offering a functional and accessible solution.

    How the System Works

    The program operates in a continuous loop, waiting for a user to press a physical button. The entire process is broken down into the following key steps:

    1. Hardware Setup and Initialization

      The script first configures the GPIO pins on the Raspberry Pi to control status LEDs and monitor the push button. It then initializes the Pi AI Camera using the picamera2 library, setting it up to capture high-resolution images. The script also sets up the credentials for Google Cloud Text-to-Speech (TTS), which is used for the voice output.

    2. Image Capture and Preprocessing

      When the user presses the button, the script captures an image from the camera. This raw image is then passed through a series of image processing techniques to enhance its quality for OCR. This includes:

      • Grayscale Conversion and Blurring: To reduce noise and simplify the image.
      • Thresholding: To make the text stand out by converting the image to pure black and white.
      • Document Detection and Perspective Transform: An advanced step where the script tries to find the four corners of a document and corrects its perspective, as if viewed from directly above.
      • Deskewing: The image is rotated to straighten any tilted text lines, which significantly improves OCR accuracy.

    3. Text Recognition (OCR)

      The preprocessed image is fed into the Tesseract OCR engine via the pytesseract library. The script intelligently adjusts the Page Segmentation Mode (PSM) for Tesseract based on the image dimensions, using different modes for single lines of text versus full blocks of text to maximize accuracy. It performs a basic language detection check to determine whether the text is in Thai or English.

    4. Text-to-Speech (TTS) Conversion and Audio Playback

      The recognized text is sent to the Google Cloud Text-to-Speech API. The API generates an audio file of the text in the appropriate language (Thai or English), which is saved temporarily. Finally, the script uses the aplay command-line utility to play the audio file through the speaker, which is connected to the Raspberry Pi via an I2S DAC (like the MAX98357). This provides a clear, high-quality audio output.
